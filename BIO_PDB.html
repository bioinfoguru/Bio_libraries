Working with PDB files#

PDB format#

Biomolecular structures can be saved in different formats such as PDB, mmCIF, XML, etc. The PDB format is the most common format and is supported by all visualization and docking programs. The PDB class of Bio library has a PDBParser() function to instantiate a PDB parser object.

The PDBParser() function in Bio.PDB instantiates an object that have functions to read structure files. The get_structure() function takes two arguments - an id and a file name (or an open filehandle).

from Bio.PDB import *

parser = PDBParser()
structure = parser.get_structure("Protease", "7jtl.pdb")

Structure object#

The structure object has a hierarchical orgranization which can be thought of as a nested list (list of lists).

Structure \(\longrightarrow\) Model \(\longrightarrow\) Chain \(\longrightarrow\) Residue \(\longrightarrow\) Atom

Each object in this hierarchy has certain functions associated with it to get information about that object.

#for model in structure:
#    print(model)
for model in structure:
    for chain in model:
        print(chain)

To convert the sequence in three letter code to single letter code, we can use SeqUtils package available in biopython. The seq1() function returns single letter sequence for a three letter sequence. Similarly, to convert single letter sequence to three letter we have seq3() function.

from Bio.SeqUtils import seq1
seq1(prot_seq)

'QECSL'

In case the structure has some modified residues then custom mapping of three letter code to single letter code can be done using the custom_map keyword argument to seq1() function. E.g., let say a protein has a phosphorylated serine residue for which the three letter code is SEP and there is no standard single letter code for this. We can add custom mapping for SEP to S as follows.

prot_seq+="SEP"
print(seq1(prot_seq)) #without custom map
print(seq1(prot_seq,custom_map={"SEP": "S"}))

QECSLX
QECSLS

Shortcuts for accessing structure elements#

The Selection() class in Bio.PDB has some useful functions to select specific data from a structure object. E.g. the unfold_entities() function provides a shortcut to unpack a structure object at the desired level within the structure hierarchy.

atom_list = Selection.unfold_entities(structure, "A")
all_coords = [a.coord for a in atom_list]
print(all_coords[0:3])

Distance Calculations#

We can use the - operator with two atoms to get the distance between the atoms.

ca_3 = res_list[3].get_list()[1]
ca_15 = res_list[15].get_list()[1]

distance = ca_3 - ca_15
print(distance)

10.335114

Calulating Ca-Ca distance matrix

atom_list = res_list = Selection.unfold_entities(structure, "A")
ca_ca_dist = []

print("Total number of atoms", len(atom_list))
ca_atom_list = [x for x in atom_list if "CA" in x.name]
print("Number of Ca atoms", len(ca_atom_list))

for x in range(len(ca_atom_list)):
    for y in range(x + 1,len(ca_atom_list)):
        ca_ca_dist.append(ca_atom_list[x]-ca_atom_list[y])
print("Total number of Ca-Ca distances", len(ca_ca_dist))

Number of Ca atoms 203

Next we need to convert the list of Ca-Ca distance to a matrix of size 203x203. The squareform() function in the scipy library can be used for this. Once the matix is prepared, we can plot the heatmap for the distances. Also, the contact map can be generated by appling a distance cutoff.

from scipy.spatial.distance import squareform
dist_mat = squareform(ca_ca_dist)
print(dist_mat.shape)

To find contact residues based on distance cutoff, NeighborSearch() can be used. We frst need to create an atom selection that has the desired residues for which contacting residues are required. E.g., to create an atom selection having all atoms in a chain, we can use the unfold_entities() as below. Next we’ll create a NeighborSearch object having all the atoms in the other chain. Then we’ll find the residues contacting both the chains.

A_atoms = Selection.unfold_entities(structure[0]["A"], "A") #entity list, target level (A,R,C,M,S)
B_atoms = Selection.unfold_entities(structure[0]["B"], "A")

ns = NeighborSearch(A_atoms)

contacts_B = []

for atom in B_atoms:
    close_atoms = ns.search(atom.coord, 3)

Binana#

BINANA (BINding ANAlyzer) is a python library that can be used for protein-ligand binding analysis. The source code is available for download. After download the file, unzip the archive and add the path of the location where binana is there to the sys.path as shown below. You should now be able to import binana.

import sys
sys.path.append("<full_path>/binana-2.1/python/")#replace <full_path> with actual path.
import binana

# pdbqt files are available in examples dir within binana
ligand, receptor = binana.load_ligand_receptor.from_files("ligand.pdbqt", \
                                                          "receptor.pdbqt")

The interactions class has a set of functions for retriving different types of inter-molecular interactions. E.g., getting a list of hydrogen bonds between the receptor and the ligand using get_hydrogen_bonds().

hbond_inf = binana.interactions.get_hydrogen_bonds(ligand, receptor)

for hbond_label in hbond_inf["labels"]:
    print(hbond_label)
